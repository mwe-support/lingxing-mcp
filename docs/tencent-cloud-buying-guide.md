# 腾讯云购买指南：给第一次买云服务器的人

这份文档默认你是第一次买云服务器。  
它不会假设你已经懂下面这些词：

- CVM
- EIP
- SSH
- 安全组

目标也很简单：  
让你买到一台**适合做领星固定出口网关**的服务器，而不是一台“看起来像云电脑，但后面不好维护”的机器。

这份文档服务的对象也比较明确：

- 技术型卖家
- AI 协作用户
- 需要自己完成第一台固定出口网关落地的人

---

## 为什么这里默认推荐腾讯云

这不是说腾讯云一定比所有厂商都强，而是因为在当前这套公开方案里，它满足三件事：

1. 配置项标准，适合做固定出口网关
2. 国内团队更容易买、登录和运维
3. `CVM + EIP` 这条路径足够清晰，适合第一次落地

如果你没有现成偏好，直接按腾讯云做，最省心。

---

## 先记住：你到底要买什么

推荐默认值：

- **产品**：腾讯云 `CVM`
- **地域**：优先广州；如果你的团队更偏跨境或你明确想放香港，也可以选香港
- **机型**：`2核4G`
- **系统**：`Ubuntu 22.04 LTS`
- **系统盘**：`40GB - 50GB`
- **带宽**：基础带宽即可
- **公网出口**：长期团队共享建议绑定 `EIP`

### 广州和香港怎么选

可以这样理解：

- **广州优先**
  - 团队主要在中国大陆
  - 日常访问更追求低延迟
  - 想先尽快跑通
- **香港优先**
  - 你明确想把固定出口放在境外
  - 你更熟悉香港节点的管理方式

如果你拿不准，先选 **广州**，通常更适合作为第一台固定出口网关。

---

## 什么是 EIP，为什么推荐它

`EIP` 可以理解成“独立持有的固定公网 IP”。

如果你只是临时验证，当前实例自带公网 IP 也能先跑。  
但如果你打算长期给团队共享，`EIP` 更稳，因为：

- 机器重建时更容易保住出口 IP
- 白名单更稳定
- 不用把“这台实例当前的临时公网 IP”当长期方案

一句话：

- **先跑通**：当前公网 IP 可以先用
- **长期共享**：更推荐 `EIP`

## 去哪里买

请直接在腾讯云官网买：

- 云服务器 `CVM`
- 弹性公网 IP `EIP`

不要把“能远程桌面”误以为“适合做固定出口网关”。  
也不要默认把第三方“云设备”当成标准服务器替代品。

## 在购买页怎么选

下面按第一次购买来写，尽量只讲你必须关心的项。

### 1. 地域

优先：

- 广州

备选：

- 香港

### 2. 配置

推荐：

- `2核4G`

这套配置对领星 MCP 足够了。  
不要一上来买 `4核8G`，通常没必要。

### 3. 操作系统

必须改成：

- `Ubuntu Server 22.04 LTS`

原因很简单：当前公开脚本和部署说明默认按 Ubuntu 写的。

### 4. 系统盘

推荐：

- `40GB - 50GB`

### 5. 带宽

基础带宽即可。  
这个场景主要是 API、SSH、少量网关流量，不需要大带宽。

### 6. 登录方式

最理想：

- `SSH key`

如果购买页只能先选密码，也没关系，可以先买。  
买完登录后再把 SSH key 补上。

### 7. 安全组

初期最保守的做法：

- 只放行 `22`

先不要把 `8099` 暴露到公网。

## 买完后要记下什么

下单成功后，请至少记下：

- 公网 IP
- 登录用户名
- 实例 ID
- 当前是否已经绑定 `EIP`

这些信息后面都要用到。

---

## 买完后下一步做什么

### 第 1 步：先登录服务器

如果你用的是密码登录，就先用密码进服务器。  
登录后，再补 SSH key 也可以。

### 第 2 步：部署 HTTP gateway

按公开仓库里的部署脚本来：

- `deploy/install_gateway_on_ubuntu.sh`
- `deploy/deploy_gateway_via_ssh.sh`

### 第 3 步：确认出口 IP

你需要确认服务器真正出去的公网 IP。  
这个 IP 才是后面加到领星白名单里的值。

### 第 4 步：回到领星后台加白名单

路径：

`设置 -> 业务配置 -> 基础 -> 开放接口`

把固定出口 IP 加入白名单，再添加 API 账号，拿到 `APP ID / AppSecret`。

### 第 5 步：做最小验证

建议按这个顺序：

1. `healthz`
2. `lingxing_health_check`
3. `lingxing_smoke_check`

## 常见误区

### 误区 1：有公网 IP 就一定不需要 EIP

不完全对。

- 临时验证：可以先用当前公网 IP
- 长期团队共享：还是更推荐 `EIP`

### 误区 2：能远程桌面就适合做固定出口网关

不对。

“能远程桌面”只说明它像一台远程电脑。  
但我们更关心的是：

- 出口 IP 是否稳定
- 能不能自己装依赖
- 能不能长期托管服务
- 能不能清楚管理 SSH、安全组、环境变量

### 误区 3：第三方“云设备”一定等价于标准云服务器

不对。

如果你真正要做的是固定出口网关，标准 `CVM + EIP` 会更清晰、更可控。

### 误区 4：第一次买服务器一定要一次买到最完美

也不对。

更实用的做法是：

1. 先按默认值买一台能跑通的
2. 先把领星白名单和网关跑通
3. 再逐步补 EIP、SSH key、团队接入优化

> - 真实 `AppSecret`
> - 真实 Bearer
> - 真实 `tokens.json`
> - 真实服务器公网 IP
>
