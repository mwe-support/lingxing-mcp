# 公众号介绍草案

## 标题候选

- 领星 OpenAPI 终于能像 MCP 一样稳定接入团队工作流了
- 解决领星白名单难题：一套固定出口网关，让团队随时随地拉数据
- 我们把领星 ERP 做成了 MCP，但不是为了替代领星

## 正文草案

很多做亚马逊团队协作的人，都碰到过同一个问题：  
领星 OpenAPI 能力很强，但真正想让团队稳定调用时，最麻烦的不是接口本身，而是部署。

最常见的卡点有两个：

- 白名单 IP 数量有限
- 团队成员会在不同地点、不同网络、不同 VPN 环境下工作

这就导致一个很尴尬的局面：  
API 明明已经申请好了，权限也开了，但真正落到团队使用时，总有人因为网络出口变化而连不上。

我们这次做的，不是另造一个 ERP，也不是去替代领星后台。  
我们做的是一个**只读 LingXing MCP**：

- 把领星 OpenAPI 封成 MCP 工具
- 用固定出口网关解决白名单问题
- 用多人令牌解决团队共享问题
- 让 Claude Code、Codex、Cursor 这类工具都能稳定接入
- 把广告、流量、利润、库存、补货、报告导出这些只读能力一起收进来

它更像一层“数据接入中间层”：

- 原始经营数据还是来自领星
- AI 和 IDE 工具负责读取、分析、组织和输出
- 团队成员不需要把主密钥放在自己电脑上

而且这次不是只停留在“查销量、查订单”这一层。  
现在这套只读面已经能覆盖很多团队日常最常用的分析场景，比如：

- 店铺销量、ASIN 日报、订单、促销
- 产品表现：浏览、会话、广告、销量
- 广告报表：SP / SD / SB
- 利润视图：店铺、ASIN、父 ASIN
- 亚马逊源表、FBA 库存、补货限制
- Amazon 报告导出与文件解析
- ASIN 广告日汇总、周汇总

这意味着它不只是“把领星接给 AI 看一眼”，而是真的可以承接：

- 周报补数
- ASIN 经营分析
- 广告与自然流量联动分析
- 多店铺团队共享查询
- 需要下载 Amazon 报告再做二次分析的流程

这一套特别适合：

- 有多个运营同事同时看数据的团队
- 需要在不同网络环境下工作的人
- 想把领星数据接进 AI 工作流的人
